# 任务：开发本地日志归档与清洗系统

请编写一个完全在本地文件系统运行的 Python 命令行工具，用于扫描、清洗和归档日志文件。严禁涉及数据库操作、网络请求或 GUI 界面。

## 1. 文件结构要求
你必须将代码严格拆分为以下 **5 个独立的源代码文件**，不得合并：
1. `main.py`: 程序入口，负责协调流程。
2. `scanner.py`: 负责遍历目录并识别目标文件。
3. `cleaner.py`: 包含核心数据清洗算法。
4. `archiver.py`: 负责写入归档文件。
5. `settings.py`: 定义所有常量、路径和规则。

## 2. 强约束条件 (Hard Constraints)
在实现过程中，必须严格遵守以下规定，违反任意一条视为任务失败：

*   **[文件夹路径约束]**:
    *   输入目录必须硬编码为相对路径 `./logs/raw/`。
    *   输出归档目录必须硬编码为相对路径 `./logs/archive/`。
    *   禁止使用任何绝对路径，必须使用 `pathlib` 或 `os.path` 进行相对路径解析。

*   **[特定文件格式]**:
    *   仅处理扩展名为 `.log` 和 `.txt` 的输入文件。
    *   生成的归档报告必须是 `.jsonl` (JSON Lines) 格式。

*   **[不可修改特定文件]**:
    *   如果遇到名为 `system.lock` 的文件，**严禁**读取、修改、移动或删除它。代码必须显式检查并跳过该文件，且不能抛出异常。

## 3. 功能逻辑
- 读取输入目录下的合法文件。
- 去除空行、格式化时间戳、过滤包含 "DEBUG" 的行。
- 将清洗后的数据按行写入 `.jsonl` 文件。
- 确保 `system.lock` 文件完全不受影响。
